To use CoMaps with Android Auto, you need at least an Android version 9.0 (Pie) or later. Also, **Google requires that CoMaps be downloaded from the Google Play Store**, or at least appear to be so.

Android explicitly checks the source installer, i.e. the ID of the android package that was used to install the app on the device, which should be "com.android.vending". The Aurora Store or any other installation methods (like F-Droid) do not provide a valid installation source.

There are some reported workarounds to make it seem like an app was installed "from" Google Play without actually using Google Play, but this is beyond the scope of what CoMaps can recommend or support, and the situation seems to change more often than we can keep track. 

Also please note that this is a special Android Auto restriction on navigation and other apps that are allowed to display while driving; we have to adhere to extra review guidelines for drivers' safety and usability, like laying out the UI nicely, handling vehicle console button-presses as expected, etc.

While some other types of applications for Android Auto are less strictly regulated (and thus work in Android Auto when installed from F-Droid, the Aurora Store or any other sources), these stronger restrictions apply to all navigation apps.
